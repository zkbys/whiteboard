# Script Patterns

## Stance Preservation

Keep the user's core claim intact:

- Preserve the direction of the judgment, even when sharpening language.
- Keep named people, products, or examples only when supplied by the user or clearly framed as generic.
- If the brief is only a topic, infer a thesis conservatively and record it as `Stance basis` in `polished_voiceover.md`.
- Avoid fake certainty, unverifiable metrics, and invented personal experience.

Use this rewrite lens:

```text
user idea -> sharper conflict -> concrete example -> useful method -> memorable closing line
```

## Six-Segment Timing

For a 45-second default script:

| role | target seconds | job |
| --- | ---: | --- |
| hook | 3-6 | Hit the tension in one sentence. |
| 反常识 | 5-8 | Make the counterintuitive claim. |
| 例子 | 8-12 | Show one specific operator scene. |
| 转折 | 5-8 | Reframe the scene into the thesis. |
| 方法 | 10-15 | Give a repeatable method or checklist. |
| 结论 | 4-8 | Leave a quotable final line. |

## Business Cognition Moves

Prefer these moves for IP incubation, commercial cognition, and AI cognition:

- Cost inversion: "不是省钱，是先算时间账。"
- Tool versus method: "工具是加速器，不是判断力。"
- Operator scene: "做账号、做产品、做交付、做获客" instead of abstract success talk.
- Checklist: 2-4 actions that can be repeated weekly.
- Asset framing: convert the conclusion into an IP asset, system asset, or decision asset.

## Visual Beat Compression

Create 3-6 visual beats, not one visual object per sentence. Good beats are:

- contrast: two columns, false path versus better path
- loop: repeatable workflow
- checklist: 3 decision questions or actions
- funnel: topic -> judgment -> output
- warning: trap, red box, or crossed-out wrong assumption

Each beat should contain few words and stable object IDs so later board specs can address them.
