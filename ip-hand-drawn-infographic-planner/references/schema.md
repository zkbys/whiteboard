# Infographic Planner Schema

Use this contract for the `infographic/` folder produced by `ip-hand-drawn-infographic-planner`.

## Folder Shape

```text
infographic/
├── infographic_plan.json
├── board_specs/
│   ├── board-01.board_spec.json
│   └── board-02.board_spec.json
└── image_prompts/
    ├── board-01.prompt.md
    └── board-02.prompt.md
```

Paths inside JSON should be relative to the project package root, for example `infographic/board_specs/board-01.board_spec.json`.

## `infographic_plan.json`

Required top-level fields:

- `version`: string, use `0.1`.
- `source`: object with `polishedVoiceoverPath`, `voiceoverSegmentsPath`, and `visualBeatsPath` when available.
- `boardDecision`: object explaining board count with `boardCount`, `reason`, and `splitRule`.
- `styleBridge`: object naming `hand-drawn-infographic-creator` and the prompt rules being used.
- `boards`: non-empty list of board summaries.

Each board summary must include:

- `id`: stable id such as `board-01`.
- `title`: short board title.
- `purpose`: one-sentence visual job.
- `contentDensity`: must be `simple`.
- `maxKeyObjects`: integer from 3 to 5.
- `sourceSegments`: list of voiceover segment ids.
- `diagramType`: one of `process_flow`, `comparison`, `two_panel_comparison`, `timeline`, `knowledge_map`, `checklist_flow`, or `flywheel`.
- `keyObjects`: list of compact key objects.
- `imagePromptPath`: prompt path.
- `boardSpecPath`: board spec path.

Each key object must include `id`, `label`, and `role`. Keep labels short enough to fit an information graphic; avoid full voiceover sentences.

## `board_specs/*.board_spec.json`

Board specs are semantic planning files, not animation manifests.

Required fields:

- `id`: same id as the plan board.
- `title`: same or tighter title.
- `diagramType`: same diagram type as the plan board.
- `canvas`: include `{ "width": 3400, "height": 1900, "aspectRatio": "16:9" }` unless the project has a different global canvas.
- `style`: include hand-drawn palette and color rules.
- `layout`: semantic zones and hierarchy, without exact pixel coordinates.
- `keyObjects`: objects with stable ids, labels, roles, visual forms, and source segments.
- `relationships`: optional list describing flow, contrast, or dependency.
- `imagePromptPath`: path to matching prompt.
- `handoff`: name the next Skills: `hand-drawn-infographic-creator` and `hand-drawn-infographic-video-board`.

Do not include control-layer keys in board specs:

```text
bbox
camera
cursor
annotations
boxBounds
circleCenter
underlineStart
underlineEnd
strikeStart
strikeEnd
```

Those belong to `board_manifest.json`, generated by the later control-layer Skill.

## `image_prompts/*.prompt.md`

Each prompt file must include:

- The exact sentence: `内容简洁一点，不要逐字写满口播`.
- `continuous line art`.
- `engineer's notebook sketch`.
- `whiteboard explanation aesthetic`.
- Parchment background `#faf8f3`.
- Charcoal line color `#1a2332`.
- Ocean-blue annotations `#2d5a7b`.
- No more than 1-2 semantic highlight colors.
- A negative prompt including `photorealistic`, `3D render`, and `stock photo`.

## Stable ID Rules

Use lowercase letters, digits, underscores, and hyphens only:

```text
good: workflow_asset, ai_leverage, board-01
bad: leftBox1, 右侧模块, item 1, box/ai
```

Ids should describe the concept, not the layout position.
