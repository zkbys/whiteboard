# Business Cognition Diagram Patterns

Use these patterns for IP incubation, business cognition, and AI cognition shorts.

## Board Count Decision

Default to one board unless there is a real conceptual split.

Use one board when:

- The voiceover has one thesis and one supporting contrast.
- There are 3-5 key objects total.
- The board can be understood in one glance.

Use two boards when:

- The first half frames a mistake and the second half gives a method.
- A single board would need more than 5 key objects.
- The voiceover shifts from example to reusable checklist.

Use three boards only when:

- The script is longer than the normal 30-60 second package.
- There are three independent concepts that would fight for attention.
- The user explicitly asks for a multi-board sequence.

## Diagram Types

### `comparison`

Use for false belief versus real mechanism.

Common objects:

- mistaken assumption
- hidden cost
- better alternative
- takeaway

### `two_panel_comparison`

Use when the audience must see two operating modes side by side.

Common objects:

- left problem mode
- right leverage mode
- dividing line
- bottom takeaway

### `process_flow`

Use for cause-effect or step-by-step judgment.

Common objects:

- input
- filter
- decision
- output
- feedback loop

### `checklist_flow`

Use for business rules that should become reusable decisions.

Common objects:

- question 1
- question 2
- question 3
- final decision

### `knowledge_map`

Use for a compact mental model with branches.

Common objects:

- center thesis
- 3-4 branches
- risk marker
- action marker

### `timeline`

Use only when time order is the core explanation.

Common objects:

- phase 1
- phase 2
- inflection point
- final state

### `flywheel`

Use for compounding IP or workflow loops.

Common objects:

- input source
- production action
- feedback
- reusable asset

## Compression Rules

- Convert a full sentence into a phrase of 2-10 Chinese characters when possible.
- Prefer nouns and verbs over slogans.
- Keep only what a viewer needs to follow the spoken explanation.
- Use a bottom takeaway only when it is shorter than one spoken sentence.
- Put nuance in voiceover, not in the image.

## Business/AI Cognition Object Roles

Use these roles consistently:

- `problem`: mistake, trap, hidden cost, risk.
- `alternative`: better route or mature solution.
- `mechanism`: why the result happens.
- `standard`: decision rule, scoring criterion, checklist item.
- `asset`: reusable workflow, template, Skill, or accumulated IP.
- `takeaway`: the one sentence the viewer should remember.
